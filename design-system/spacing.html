Spacing & Surfaces

A 4-step radius scale and a base-4 spacing scale — both narrow on purpose. The mobile dashboard packs four stat cards, a chart, a holdings list, and a tab bar into a 390 pt screen, so each token has to earn its width.

Spacing

Base-4 increments, biased to the values that actually appear in the codebase. Padding inside a card is almost always 24, gap between stat cards is 8, gap between sections is 16.

Radius

Mostly soft. --r-xl (16) is the default card; --r-md (8) is for inputs. Pills use --r-pill. We don't use sharp 0-radius surfaces — the design feels approachable, not industrial.

Anatomy of a Card

The default surface. Twenty-four pixels of padding inside a 16-pixel radius, separated from siblings by a hairline border (never a shadow on same-tier cards).

  • Card surface uses --card (#16161D), one step above the page.
  • Border is always 1 px --border. On hover or active, swap to --border-active; never bump width.
  • Padding is 24 on the four sides at desktop, 14 px headers + 14 px body on dense mobile lists.
  • Internal gap between label and value is 3 px — the label is small enough to feel attached.
  • Sub-stat sits 2 px below the value in --text-muted. Never centered, never italic.
  • Active card swaps the left border to a 3 px --accent stripe (used in HoldingCard when expanded).

Density

Two list densities. Comfortable is the default holdings card: 12 px row padding, 44 px minimum hit target. Tight is reserved for read-only summary tables.

Elevation

Shadows are only for true overlays — popovers, tab-bar drop, sheets. On-page cards stack via surface tint + border, not shadow. This keeps the dark UI from looking smudgy.

Resting card
No shadow. Borders + surface tint do the lifting.

Popover
